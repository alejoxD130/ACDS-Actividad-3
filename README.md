# 📦 Proyecto: Sistema de Inventario

## 🧠 Objetivo

Este ejercicio evalúa dos cosas principales:

1. Capacidad para **identificar errores y malas prácticas en el código**
2. Uso correcto de **flujo de ramas en Git**

---

## 🛠️ Qué debes hacer

- Analizar el código entregado  
- Detectar y corregir problemas  
- Mantener el sistema funcionando  

---

## 🌿 Manejo de ramas (obligatorio)

Debes trabajar usando el siguiente flujo:

- `main`: versión estable  
- `develop`: base de trabajo  

### Tipos de ramas

- **hotfix/**  
  - Para errores críticos  
  - Se crean desde `main`  
  - Se integran a `main` y `develop`  

- **feature/**  
  - Para mejoras o cambios grandes  
  - Se crean desde `develop`  
  - Se integran a `develop` mediante Pull Request  

- **fix/**  
  - Para correcciones menores (no críticas)  
  - Se crean desde `develop`  
  - Se integran a `develop` mediante Pull Request  

---

## 📌 Entregables

- Código corregido  
- Uso correcto de ramas (historial en Git)  
- Pull Requests claros  

---

## 💡 Enfoque

- Prioriza errores que afecten resultados  
- Mantén cambios pequeños y claros  
- Usa ramas correctamente según el tipo de cambio
